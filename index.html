Using Parametric Formulas.

Using Cartesian Equations.

The inner most planet follows an eliptical orbit.
The original problem I had with this when using a simple nonparametric forumula
was when it reached the boundary point (where Y is 0 and the equation flips)
is that there is a point when the y value returned can't exist, and the equation spits out an Integer.Max.
The problem with this is it would glitch around the boundaries, when it would teleport to the max Y our game supports.

The solution to this problem was to use parametric formula instead,
using maths to make X and Y dependent on the other variable of delta Time, instead of themselves.
The result is a smooth orbit that does not vary in speed - exactly what we want.

The second planet in our solar system follows a circular orbit.
Additionally, it is noticeably rotating at an angle.
This was particularly difficult to do, however in the end I managed to rotate it's parent class,
and then only update this planet's relative position, not absolute.
It uses a parametric formula, as it suffered from the problems that the eliptical planet had.
For instance, it used to speed up towards the equations boundary, like a giant booster is applied.

The second planet in our solar system follows a circular orbit too.
The main difference with this planet is the orbit is calculated by Unreal's Rotating Movement Component.
Using this, we have a comparison from in engine library, to my code.
There is little to no difference now that the other planets use parametric functions to calculate their next coordinates.
